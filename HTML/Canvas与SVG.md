# Canvas 与 SVG 面试笔记

## 1. Canvas 和 SVG 的区别

### 问题

Canvas 和 SVG 有什么区别？

### 回答

- Canvas 是位图，依赖 JavaScript 绘制
- SVG 是矢量图，基于 XML 描述图形

Canvas 更适合频繁重绘，SVG 更适合静态图标和可交互图形。

---

## 2. Canvas 适合什么场景

### 问题

Canvas 适合哪些场景？

### 回答

适合：

- 游戏
- 粒子动画
- 数据可视化中的高频绘制
- 图像处理

---

## 3. SVG 适合什么场景

### 问题

SVG 适合哪些场景？

### 回答

适合：

- 图标
- Logo
- 地图
- 流程图
- 需要缩放不失真的图形

---

## 4. 哪个更利于事件处理

### 问题

Canvas 和 SVG 哪个更方便绑定事件？

### 回答

SVG 更方便。因为 SVG 元素本身就在 DOM 树里，可以直接绑定事件；Canvas 绘图后只是像素区域，本身没有独立 DOM 节点。

## 5. 速记版

- Canvas：位图，适合高频绘制
- SVG：矢量图，适合图标和可交互图形
- SVG 缩放不失真
- Canvas 性能通常更适合复杂动画

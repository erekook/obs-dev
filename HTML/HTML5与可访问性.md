# HTML5 与可访问性面试笔记

## 1. HTML5 和可访问性有什么关系

### 问题

HTML5 为什么有助于可访问性？

### 回答

因为 HTML5 提供了更明确的结构和语义，屏幕阅读器等辅助技术可以更好地理解页面。

---

## 2. 哪些写法有利于可访问性

### 问题

前端开发中哪些 HTML 写法更有利于可访问性？

### 回答

- 使用语义化标签
- 表单使用 `label`
- 图片补充 `alt`
- 按钮用 `button`，不要滥用 `div`
- 标题层级清晰

---

## 3. ARIA 和语义化标签的关系

### 问题

ARIA 和语义化标签是什么关系？

### 回答

优先使用原生语义化标签。只有在原生标签无法满足时，再用 ARIA 做补充，而不是反过来。

## 4. 速记版

- HTML5 语义化有助于无障碍
- 优先原生标签，再考虑 ARIA
- `label` `alt` `button` 都是高频点

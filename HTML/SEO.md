## 1. 什么是 SEO

### 问题

什么是 SEO？

### 回答

SEO 是 Search Engine Optimization，也就是搜索引擎优化。目标是让网站更容易被搜索引擎抓取、理解和收录，并在相关搜索结果里获得更好的自然排名。

前端在 SEO 里主要负责三类事情：

- 页面结构是否清晰
- 页面内容是否能被搜索引擎读取
- 页面性能和可访问性是否达标

---

## 2. SEO 的作用是什么

### 问题

SEO 有什么作用？

### 回答

SEO 的核心作用是获取自然流量，降低获客成本，提升网站曝光和转化。

常见价值有：

- 提升搜索结果排名
- 增加自然流量
- 提高品牌曝光
- 提升页面点击率和转化率

---

## 3. 前端为什么要关注 SEO

### 问题

SEO 不是运营做的吗？前端为什么要关注？

### 回答

不是只有运营才关心 SEO。很多 SEO 基础能力直接由前端实现，比如：

- 是否使用语义化标签
- 页面是否有正确的 title 和 meta description
- 是否有合理的 heading 结构
- 页面内容是否能在首屏 HTML 中被搜索引擎拿到
- 页面加载速度是否足够快
- 是否支持移动端

如果前端实现不合理，即使内容本身不错，也可能影响收录和排名。

---

## 4. SEO 常见优化方式有哪些

### 问题

SEO 常见优化方式有哪些？

### 回答

一般可以分为这几类：

- 内容优化：关键词布局、内容质量、搜索意图匹配
- 结构优化：title、description、heading、内链
- 技术优化：SSR、静态化、canonical、sitemap、robots
- 性能优化：首屏速度、图片优化、缓存、减少阻塞资源
- 站外优化：高质量外链、品牌提及

前端面试里最常问的是结构优化、技术优化和性能优化。

## 5. SEO 中常见的 meta 标签有哪些

### 问题

SEO 中常见的 meta 标签有哪些？

### 回答

常见的有：

- `title`：页面标题，影响搜索结果标题展示
- `meta name="description"`：页面描述，影响摘要展示和点击率
- `meta name="keywords"`：现在主流搜索引擎基本不再依赖
- `meta name="robots"`：控制索引和抓取行为
- `link rel="canonical"`：指定规范地址，避免重复内容

面试时可以补一句：

现在 `keywords` 的 SEO 价值很低，真正重要的是 title、description、canonical 和页面内容本身。

## 6. title 和 description 应该怎么优化

### 问题

title 和 meta description 应该怎么写？

### 回答

优化原则：

- 每个页面都要唯一
- title 要准确概括页面主题
- 核心关键词尽量自然出现
- description 要概括页面价值，提升点击率
- 不要堆砌关键词

示例：

```html
<title>前端 SEO 面试题总结</title>
<meta name="description" content="整理前端面试中常见的 SEO 问题，包括语义化标签、SSR、meta 标签、性能优化等内容。">
```

## 7. 语义化标签为什么有利于 SEO

### 问题

为什么说语义化标签有利于 SEO？

### 回答

语义化标签可以让搜索引擎更容易理解页面结构和内容层级。

常见标签有：

- `header`
- `nav`
- `main`
- `article`
- `section`
- `aside`
- `footer`

相比大量 `div`，语义化标签能更清楚地表达哪部分是导航、正文、侧边栏和页脚，这有助于搜索引擎理解页面。

不过要注意，语义化标签不是决定排名的唯一因素，它更多是帮助理解页面结构。

## 8. H1、H2、H3 有什么作用

### 问题

标题标签在 SEO 中有什么作用？

### 回答

标题标签用于表达页面内容层级，帮助搜索引擎理解页面结构。

常见原则：

- 一个页面通常只有一个 H1
- H1 表示页面主主题
- H2 表示主要章节
- H3 表示子章节

不要为了样式随便使用标题标签，应该按照内容层级来写。

## 9. 什么是 SSR？为什么 SSR 对 SEO 更友好

### 问题

为什么说 SSR 比 CSR 更利于 SEO？

### 回答

SSR 是服务端渲染，服务器先生成完整 HTML，再返回给浏览器。CSR 是客户端渲染，浏览器先拿到一个空壳 HTML，再通过 JavaScript 渲染页面内容。

SSR 对 SEO 更友好的原因：

- 搜索引擎更容易直接拿到完整内容
- 首屏内容更快可见
- 对爬虫更稳定
- 降低因 JS 执行失败导致内容缺失的风险

但也要注意：

- 现在主流搜索引擎可以执行 JavaScript
- 不是说 CSR 一定不能做 SEO
- 只是 SSR、SSG 对内容型站点通常更稳

## 10. SSR、CSR、SSG 的区别

### 问题

SSR、CSR、SSG 有什么区别？SEO 上怎么选？

### 回答

区别：

- CSR：页面主要在浏览器端渲染
- SSR：每次请求由服务端渲染 HTML
- SSG：构建时提前生成静态 HTML

SEO 角度：

- 内容相对稳定的页面，优先考虑 SSG
- 需要实时数据的页面，可以用 SSR
- 纯后台系统或不依赖搜索流量的页面，CSR 也可以

面试中可以总结为：

如果项目对 SEO 要求高，通常优先考虑 SSR 或 SSG。

## 11. 单页应用怎么做 SEO

### 问题

SPA 怎么做 SEO？

### 回答

单页应用天然对 SEO 不够友好，因为很多内容依赖客户端渲染。

常见解决方案：

- 使用 SSR 或 SSG 框架，比如 Next.js、Nuxt
- 给不同页面设置独立的 title 和 description
- 做预渲染
- 保证路由可访问且有真实 URL
- 提供 sitemap.xml

如果是 React 项目，面试时常见回答是：

可以通过 Next.js 做 SSR 或静态生成，提高页面可抓取性。

## 12. 什么是预渲染

### 问题

什么是预渲染？

### 回答

预渲染就是在构建阶段或请求前先生成页面 HTML，让搜索引擎和用户一开始就能拿到完整内容。

它适合：

- 页面内容变化不频繁
- 营销页
- 博客
- 文档站

它本质上是提升 SEO 友好度的一种方式。

## 13. 图片怎么做 SEO 优化

### 问题

图片相关的 SEO 优化有哪些？

### 回答

常见做法：

- 给图片添加有意义的 `alt`
- 文件名尽量语义化
- 压缩图片体积
- 使用合适格式，比如 WebP
- 对首屏重要图片做懒加载策略权衡

其中 `alt` 很常被面试官问到，因为它既影响可访问性，也有助于搜索引擎理解图片内容。

示例：

```html
<img src="/images/seo-guide.webp" alt="前端 SEO 优化思维导图">
```

## 14. canonical 是什么

### 问题

什么是 canonical？有什么作用？

### 回答

canonical 用来告诉搜索引擎，多个相似或重复页面中，哪一个才是规范页面。

作用：

- 避免重复内容问题
- 集中权重
- 防止参数 URL、分页 URL 分散排名

示例：

```html
<link rel="canonical" href="https://example.com/html/seo">
```

## 15. robots.txt 和 meta robots 有什么区别

### 问题

robots.txt 和 meta robots 有什么区别？

### 回答

区别在控制层级不同：

- `robots.txt` 是站点级规则，主要控制爬虫能不能抓取某些路径
- `meta robots` 是页面级规则，主要控制页面能不能被索引、能不能跟踪链接

示例：

```html
<meta name="robots" content="index,follow">
```

常见误区：

`Disallow` 不等于一定不会被索引，因为某些 URL 即使不能抓取，也可能因外部链接而被发现。

## 16. sitemap.xml 有什么作用

### 问题

sitemap.xml 有什么作用？

### 回答

sitemap 是网站地图，用来告诉搜索引擎网站有哪些重要页面，方便爬虫发现和抓取。

它的作用：

- 帮助新页面更快被发现
- 帮助大型网站组织 URL
- 对动态站点或层级较深的网站更有帮助

不过 sitemap 只是辅助抓取，不代表提交后一定会收录。

## 17. 页面性能为什么会影响 SEO

### 问题

为什么说页面性能也会影响 SEO？

### 回答

因为搜索引擎会关注用户体验，页面越慢，通常跳出率越高，体验越差。

前端常见优化点：

- 减少首屏资源体积
- 压缩图片和静态资源
- 使用缓存
- 代码分割
- 减少阻塞渲染的 JS 和 CSS
- 优化 Core Web Vitals，比如 LCP、CLS、INP

面试时如果能提到 Core Web Vitals，会更完整。

## 18. SEO 和可访问性有什么关系

### 问题

SEO 和可访问性有关吗？

### 回答

有关系。SEO 和可访问性并不完全等同，但很多最佳实践是重合的。

例如：

- 合理的语义化标签
- 正确的 heading 结构
- 图片 `alt`
- 清晰的链接文本

这些做法既帮助搜索引擎理解页面，也帮助辅助技术理解页面。

## 19. nofollow、nofollow 链接有什么作用

### 问题

`nofollow` 是什么？有什么用？

### 回答

`nofollow` 表示告诉搜索引擎不要跟踪某个链接，常用于不想传递权重或者用户生成内容场景。

示例：

```html
<a href="https://example.com" rel="nofollow">示例链接</a>
```

不过现在搜索引擎对 `nofollow` 的处理更偏向提示，而不是绝对指令。

## 20. 面试中如果让我回答“前端如何做 SEO”，可以怎么总结

### 问题

前端如何做 SEO？

### 回答

可以从这几个方面总结：

1. 使用语义化 HTML，保证页面结构清晰
2. 合理设置 title、description、canonical、robots 等标签
3. 正确使用 H1-H6、图片 alt、内链结构
4. 对需要搜索流量的页面采用 SSR、SSG 或预渲染
5. 提升页面性能和移动端体验
6. 配置 sitemap.xml 和 robots.txt
7. 避免重复内容、空白首屏和无意义 URL

如果要用一句话概括：

前端做 SEO，本质上是在保证页面内容可抓取、结构可理解、访问体验足够好。

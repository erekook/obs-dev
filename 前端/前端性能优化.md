## 页面性能指标（和我们怎么写代码有关）
- 首屏渲染时：fcp（first content paint）页面第一个元素的渲染时长
	- 懒加载：需要写代码实现
	- http优化：协商缓存和强缓存
		- 强缓存：服务端给响应头追加一些字段（expires），客户端会记住这些字段，在expires失效时间内，无论刷新页面，都会从缓存里获取
		- 协商缓存：是否使用缓存要和后端商量一下，当服务端给我们打上协商缓存的标记后，客户端下次刷新页面需要重新请求资源时会发送一个协商请求给服务端，服务端如果说需要变化，则会响应具体的内容，如果服务端觉得没有变化，则会响应304
- 页面中最大的元素的一个时长：lcp（largest content paint）
- ...

## JS逻辑

- 我们要注意副作用的清除， 组件是会频繁的挂载和卸载：如果我们在一个组件里有计时器（setTimeout），如果组件卸载时不清除计时器，下次挂载时计时器等于开了2个线程
```js
const [timer, setTimer] = useState(null)
useEffect(()=> {
	setTimer(setTimeout(()=> {
	}));
	return () => {
		clearTimeout(timer)
	}
})
```
- 我们在写法上的一个注意事项：requestAnimationFrame, requestIdleCallback 卡浏览器帧率，对浏览器原理要有一定的认识，然后再做这方面的优化
	- requestIdleCallback  传一个函数进去
	- 浏览器的帧率，16.6ms去更新一次（执行js逻辑，以及重排重绘...）假设我的js执行逻辑超过了16.6ms，掉帧，卡顿
	- cocurrent mode cocurrency 可终端渲染 react
- 防抖，节流 最好用loadsh
```js
const arr = [];// 几千条
arr.forEach // 不要用forEach，用loadsh.forEach
```

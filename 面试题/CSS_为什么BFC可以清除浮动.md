# 题目
为什么 BFC 可以清除浮动？

# 你的回答
后来追问了这个问题。

# 面试点评
这是理解 BFC 的关键点。重点不是“取消浮动”，而是“让父元素重新计算并包裹浮动子元素”。

# 回答要点
- 浮动元素脱离普通文档流
- 父元素可能无法正确计算浮动子元素高度
- 形成 BFC 后，内部浮动元素会参与高度计算
- 父元素能包裹住浮动元素，从而表现为清除浮动

# 参考答案
因为浮动元素会脱离普通文档流，父元素如果没有处理，可能无法撑开高度。
当父元素形成 BFC 后，会成为一个独立的布局区域，在计算高度时会把内部浮动元素也计算进去，所以父元素能够包裹住浮动子元素，从而达到清除浮动的效果。
需要注意的是，BFC 不是把 float 取消了，而是让父元素能正确计算浮动元素的高度。

# 题目
什么是 BFC（Block Formatting Context）？它有哪些触发方式？BFC 如何用来解决外边距塌陷和布局问题？

# 回答要点
- BFC 是块级格式化上下文，是页面中的一块独立渲染区域。
- BFC 内部元素的布局不会影响外部，外部也不会影响内部的块级布局计算。
- 常见触发方式：
  - 根元素 `html`
  - `float` 不为 `none`
  - `position` 为 `absolute` 或 `fixed`
  - `overflow` 不为 `visible`
  - `display: inline-block`、`table-cell`、`flex`、`grid` 等
- BFC 的典型特性：
  - 同一个 BFC 内，相邻块级元素垂直 margin 可能发生重叠
  - BFC 区域不会与浮动元素重叠
  - 计算 BFC 高度时，浮动子元素也会参与计算
- 解决外边距塌陷：
  - 给父元素创建 BFC，可避免父子之间的 margin 合并带来的“塌陷感”
  - 对需要隔离的兄弟块包裹不同 BFC，也能避免一些相互影响
- 解决布局问题：
  - 清除内部浮动导致的父元素高度塌陷
  - 实现两栏布局，避免文字环绕浮动错位

# 追问
- BFC 和普通文档流的区别是什么？
- 父子元素、兄弟元素的 margin 合并分别是什么情况？
- 为什么 `overflow: hidden` 经常能“修复”布局？它的副作用是什么？
- `display: flow-root` 和传统触发 BFC 的方式相比有什么优缺点？

# 参考答案
BFC 可以理解为一个独立的布局环境，里面的盒子按照块级格式化规则进行排列，外部布局不会直接干扰它，反过来它内部的布局结果也相对独立。

常见触发 BFC 的方式包括：给元素设置 `float`，设置 `position: absolute/fixed`，设置 `overflow: hidden/auto/scroll`，或者使用 `inline-block`、`flex`、`grid`、`flow-root` 等显示类型。

BFC 常见的几个用途里，面试最爱问的是两个。

第一，解决浮动导致的父元素高度塌陷。因为普通情况下，父元素在计算高度时不会把脱离普通文档流的浮动子元素完整算进去；但如果父元素形成了 BFC，那么它会把浮动子元素的高度纳入计算，这样父元素就能“撑开”。

第二，处理 margin 合并和局部布局隔离。比如父元素的第一个子元素设置了 `margin-top`，有时看起来像把父元素整体顶下去了，这本质上是父子之间发生了 margin 合并。让父元素形成 BFC 后，就能在很多场景下阻止这种影响扩散。兄弟元素之间如果需要更明确的布局边界，也可以通过 BFC 来隔离。

另外，BFC 还常用于避免内容与浮动元素重叠。比如左侧有一个浮动盒子，右侧普通块级内容可能围绕它排布；如果右侧内容形成 BFC，它会避开浮动区域，从而实现更稳定的两栏布局。

实际开发里，`overflow: hidden` 过去很常用，因为写法简单。但它可能带来裁剪溢出内容的问题，所以现在更推荐语义更清晰的 `display: flow-root` 来创建 BFC。

# 题目
请你讲一下 CSS 里 `flex: 1` 到底是什么意思。它和 `flex-grow`、`flex-shrink`、`flex-basis` 是什么关系？

# 回答要点
- `flex` 是复合属性，简写了：
  - `flex-grow`
  - `flex-shrink`
  - `flex-basis`
- `flex: 1` 常见等价理解：
  - `flex: 1 1 0%`
- 含义拆分：
  - `flex-grow: 1`，有剩余空间时可以扩张
  - `flex-shrink: 1`，空间不足时允许收缩
  - `flex-basis: 0%`，初始基准尺寸按 0 参与分配
- 为什么常用于等分布局：
  - 多个子项都 `flex: 1` 时，会按比例瓜分剩余空间
- 面试里可补充：
  - `flex: auto`、`flex: none`、`flex: initial` 都和 `flex: 1` 不同
  - `flex-basis` 对最终分配结果影响很大

# 追问
- 为什么 `flex: 1` 经常看起来像“均分剩余空间”？
- `flex: 1` 和 `width: 100%` 是一回事吗？
- `flex: auto` 和 `flex: 1` 的区别是什么？
- 为什么有时还要给 flex 子项加 `min-width: 0`？

# 参考答案
很多人写 `flex: 1` 时知道它能“自动撑开”，但如果面试里被追问含义，关键是要知道它其实是 `flex-grow`、`flex-shrink`、`flex-basis` 的简写。

在现代浏览器里，`flex: 1` 一般可以理解为：

```css
flex: 1 1 0%;
```

这里的三个值分别表示：

- `flex-grow: 1`，说明有剩余空间时，这个子项可以扩张。
- `flex-shrink: 1`，说明容器空间不够时，这个子项也允许收缩。
- `flex-basis: 0%`，表示在分配剩余空间之前，它的基础尺寸先按 0 来参与计算。

也正因为基准是 0，所以当多个子项都写 `flex: 1` 时，它们通常会比较接近“平均分配剩余空间”的效果。这也是为什么 `flex: 1` 常用于等分布局、左右自适应布局、卡片均分等场景。

不过 `flex: 1` 并不等于“永远平分”。最终结果还会受到内容本身、最小宽度、容器空间、是否允许收缩等因素影响。比如有些内容特别长时，如果不额外处理，布局可能不会像你想象得那么整齐。

和它容易混淆的还有 `flex: auto`。`flex: auto` 通常可以理解为 `1 1 auto`，也就是初始尺寸会参考元素自身内容或宽高，而不是像 `flex: 1` 那样从 0 开始参与分配。所以它们的最终表现可能不一样。

面试里如果总结，可以说：`flex: 1` 本质是 `1 1 0%`，表示元素可扩张、可收缩，并以 0 作为分配基准，因此常被用来均分剩余空间。把这句讲清楚，基本就够了。

# 题目
请你讲一下 CSS 里的 Flex 和 Grid 怎么选。它们都能做布局，但各自更适合什么场景？

# 回答要点
- Flex 和 Grid 都是现代布局方案，但关注点不同。
- Flex：
  - 一维布局
  - 更适合处理一行或一列上的排列和对齐
  - 常用于导航、按钮组、左右布局、居中
- Grid：
  - 二维布局
  - 更适合同时控制行和列
  - 常用于复杂面板、卡片矩阵、后台看板
- 选择原则：
  - 只关心主轴/交叉轴上的排列，优先 Flex
  - 需要明确网格坐标和二维区域关系，优先 Grid
- 面试里可补充：
  - 它们不是互斥关系，实际项目中常常会混用
  - 页面大结构可能用 Grid，局部组件内部用 Flex

# 追问
- 为什么说 Flex 是一维布局，Grid 是二维布局？
- Grid 为什么更适合复杂后台页面？
- 什么时候用 Grid 反而会显得过重？
- `gap` 在 Flex 和 Grid 中使用体验有何异同？

# 参考答案
Flex 和 Grid 都是现代 CSS 布局方案，但它们擅长解决的问题不完全一样。

最简单的区分方式是：Flex 更偏一维布局，Grid 更偏二维布局。

所谓一维布局，意思是你主要在处理“同一行”或者“同一列”里的排列问题。比如几个按钮怎么排、左右两栏怎么分、元素怎么水平垂直居中，这类问题通常用 Flex 最自然。它特别擅长处理元素沿主轴和交叉轴的分布、对齐和伸缩。

而 Grid 更像在处理一个真正的网格系统，也就是你同时要关心行和列。比如一个后台首页有多个信息卡片区域、左上右下都有明确位置关系，或者你想精确控制某个模块占几行几列，这种时候 Grid 的表达力通常更强。

所以如果只是做导航栏、工具栏、列表项内部布局、按钮组、左右自适应这些相对线性的布局，优先用 Flex 往往更简单；如果你面对的是更复杂的页面结构，想像搭积木一样规划二维区域，Grid 通常会更舒服。

实际开发里，它们也不是二选一关系。很常见的做法是：页面整体结构用 Grid 来分区，具体某个区块内部再用 Flex 做细节对齐。这样既保留 Grid 的结构表达力，也保留 Flex 的局部灵活性。

面试里如果总结，可以说：Flex 适合一维排列和对齐，Grid 适合二维网格布局；不是谁更高级，而是看你是在解决“排一条线”，还是“搭一个面”。

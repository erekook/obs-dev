# 题目
请你讲一下 CSS 里 `position: static / relative / absolute / fixed / sticky` 分别有什么区别，常见使用场景是什么？

# 回答要点
- `static`：
  - 默认定位方式
  - 不支持 `top/right/bottom/left`
- `relative`：
  - 相对自己原本位置偏移
  - 仍保留原来的占位
  - 常作为绝对定位子元素的参照
- `absolute`：
  - 脱离普通文档流
  - 相对于最近的非 static 祖先定位
  - 常用于弹层、角标、局部浮动元素
- `fixed`：
  - 脱离文档流
  - 相对于视口定位
  - 常用于吸附按钮、固定导航
- `sticky`：
  - 在达到阈值前像普通流中的元素
  - 达到阈值后像固定定位一样吸附
  - 常用于列表标题吸顶
- 面试里可补充：
  - `absolute/fixed` 脱离文档流会影响布局占位
  - `sticky` 依赖滚动容器和阈值条件

# 追问
- 为什么 `absolute` 要找最近的非 static 祖先？
- `fixed` 在某些复杂容器里为什么会出现“看起来不像固定在视口”的情况？
- `sticky` 为什么有时不生效？
- `relative` 常常为什么配合 `absolute` 一起用？

# 参考答案
CSS 里的 `position` 决定了元素如何参与布局，以及它偏移时参考谁。

`static` 是默认值，也就是正常文档流布局。元素按顺序排下去，`top/right/bottom/left` 这些偏移属性对它通常不起作用。

`relative` 是相对定位。元素仍然保留在原来的文档流位置上，只是视觉上相对自己原本位置做偏移，所以它“占位还在，人可以挪一下”。它还有一个非常常见的用途，就是给子元素的绝对定位提供参考系。

`absolute` 是绝对定位。它会脱离普通文档流，不再占据原来的布局位置。它的位置通常相对于最近的一个 `position` 不是 `static` 的祖先元素来计算。如果找不到，就可能相对更外层的容器甚至视口。

`fixed` 也是脱离文档流的，但它通常相对于浏览器视口定位。页面滚动时，它不会跟普通内容一起移动，所以很适合固定导航、悬浮按钮、回到顶部这类场景。

`sticky` 可以理解成“粘性定位”。它在达到某个滚动阈值前，表现得像普通流里的元素；滚动到指定位置后，会像固定定位一样吸附住。常见场景是通讯录分组标题吸顶、表头吸顶等。

如果从使用场景来总结：普通布局默认 `static`；轻微偏移和定位上下文常用 `relative`；局部浮层和角标常用 `absolute`；固定在视口上的元素常用 `fixed`；随滚动吸附常用 `sticky`。

面试里如果总结，可以说：`relative` 是“保留占位的相对偏移”，`absolute/fixed` 是“脱离文档流的定位”，`sticky` 是“滚动过程中的条件性固定”，理解它们和文档流、参考系的关系最关键。

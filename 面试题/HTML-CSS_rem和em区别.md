# 题目
请你讲一下 CSS 里的 `rem` 和 `em` 有什么区别。它们各自是相对于谁计算的，适合什么场景？

# 回答要点
- 两者都是相对长度单位。
- `rem`：
  - 相对于根元素 `html` 的 `font-size`
  - 全局一致性更强
- `em`：
  - 相对于当前元素的 `font-size`
  - 如果用于 `font-size` 本身，通常相对于父元素字体大小
  - 会受到嵌套层级影响
- 区别核心：
  - `rem` 更适合做全局统一缩放
  - `em` 更适合局部相对缩放
- 常见场景：
  - `rem`：响应式布局、统一尺寸体系
  - `em`：按钮内边距、局部组件跟随字体缩放
- 面试里可补充：
  - 深层嵌套下 `em` 可能带来累积放大/缩小问题
  - 现代项目里很多布局更常见 `rem + px` 或 `rem + vw` 配合使用

# 追问
- 为什么 `em` 在嵌套场景里容易让尺寸失控？
- 如果用户修改浏览器默认字号，`rem` 和 `em` 会怎样变化？
- `rem` 在移动端适配里通常怎么配合使用？
- 为什么有些场景仍然更推荐 `em` 而不是 `rem`？

# 参考答案
`rem` 和 `em` 都是 CSS 里的相对单位，但它们参照的对象不一样。

`rem` 是 root em，顾名思义，它始终相对于根元素，也就是 `html` 的 `font-size` 来计算。比如如果根元素字体大小是 16px，那么 `1rem` 就等于 16px，`2rem` 就等于 32px。不管元素嵌套多深，这个参照都不会变。

`em` 则是相对于当前元素的字体大小来计算的。如果它用在普通属性上，比如 `padding`、`width`，一般是参考当前元素自己的 `font-size`；如果它本身就用在 `font-size` 上，通常会相对于父元素字体大小来算。

这就导致两者最大的区别在于稳定性。`rem` 更适合建立全局统一的尺寸体系，因为整个页面的基准都跟着根元素走，控制起来更一致。`em` 更适合局部相对缩放，比如希望按钮的内边距随着按钮字体变化而一起变化，这时候 `em` 会比较自然。

但 `em` 也有一个常见问题，就是嵌套层级深时容易出现累积效应。比如父元素用了 `font-size: 1.2em`，子元素再写 `1.2em`，层层叠加后尺寸就可能越来越大或越来越小，不太容易控制。

所以实际开发里，如果是页面级布局、统一间距、响应式尺寸体系，通常更倾向用 `rem`；如果是局部组件内部，希望尺寸跟随当前文字比例变化，`em` 会更合适。

面试里可以一句话总结：`rem` 是相对根元素，适合全局统一；`em` 是相对当前上下文字体，适合局部联动缩放。把这句讲清楚，基本就够了。

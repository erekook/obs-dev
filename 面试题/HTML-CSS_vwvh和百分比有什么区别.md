# 题目
请你讲一下 CSS 里的 `vw`、`vh` 和百分比 `%` 有什么区别。它们分别是相对于谁计算的，适合什么场景？

# 回答要点
- `vw` / `vh` 是视口单位：
  - `1vw` = 视口宽度的 1%
  - `1vh` = 视口高度的 1%
- `%` 是相对单位，但具体参照对象取决于属性和上下文：
  - 可能相对于父元素宽度
  - 也可能相对于父元素高度
  - 某些属性还可能有其他计算规则
- 核心区别：
  - `vw/vh` 直接相对浏览器视口
  - `%` 通常相对父级或包含块
- 常见场景：
  - `vw/vh`：全屏布局、视口相关适配、首屏区块
  - `%`：父子布局、自适应宽高、流式布局
- 面试里可补充：
  - 移动端 `vh` 在浏览器地址栏伸缩场景下可能有坑
  - 新单位如 `svh`、`lvh`、`dvh` 是相关延伸

# 追问
- 为什么设置 `height: 100%` 有时不生效？
- `100vh` 在移动端为什么可能出现视觉高度问题？
- `vw` 做字体适配有什么优缺点？
- 父元素没有明确高度时，子元素 `height: 100%` 为什么常常没用？

# 参考答案
`vw`、`vh` 和百分比 `%` 都是相对单位，但它们参照的对象并不一样。

`vw` 和 `vh` 是视口单位，直接相对于浏览器可视区域来算。`1vw` 表示视口宽度的 1%，`1vh` 表示视口高度的 1%。所以如果浏览器窗口宽 1000px，那么 `50vw` 就是 500px。

而 `%` 则不是固定相对视口，它通常是相对于父元素或包含块来计算的。不过要注意，具体参照谁，要看你写的是哪个属性。比如 `width: 50%` 往往相对父元素宽度；但 `height: 100%` 是否生效，则取决于父元素有没有明确高度。

所以它们最核心的区别可以概括为：`vw/vh` 直接跟视口绑定，适合做全屏区块、首屏 Banner、视口相关布局；而 `%` 更适合做父子之间的比例布局，比如容器占父元素一半宽度、自适应子区域等。

实际开发里，如果你要做“铺满整个屏幕”的区域，通常会想到 `100vw` 或 `100vh`；如果你要让子元素跟着父元素尺寸变化，更自然的通常是 `%`。

不过 `vh` 在移动端有一个常见坑。因为浏览器地址栏、底部工具栏会动态伸缩，导致 `100vh` 在某些时刻看起来比真实可见区域更高或更低，所以现代项目里有时会结合 `dvh`、`svh` 这类新单位，或者用 JS 做补充处理。

面试里如果总结，可以说：`vw/vh` 是相对视口的全局比例单位，`%` 是相对父级或包含块的局部比例单位；前者更适合屏幕级适配，后者更适合容器级布局。

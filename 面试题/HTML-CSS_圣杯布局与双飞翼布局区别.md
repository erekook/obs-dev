# 题目
请你讲一下圣杯布局和双飞翼布局。它们分别是怎么实现的？核心区别是什么？

# 回答要点
- 两者都用于实现“三栏布局”：中间自适应，两侧固定宽度。
- 设计目标：
  - 中间内容优先渲染
  - 左右栏固定宽度
  - 中间栏自适应剩余空间
- 圣杯布局思路：
  - HTML 结构通常先写中间栏，再写左右栏
  - 父容器预留左右 padding
  - 三栏浮动
  - 左右栏通过负 margin 和相对定位回到两侧
- 双飞翼布局思路：
  - 也是三栏浮动
  - 中间栏外层占满宽度
  - 中间内容再包一层，内部设置左右 margin 给两侧留空间
  - 左右栏通过负 margin 拉回两侧
- 核心区别：
  - 圣杯布局通过父容器 padding 预留空间
  - 双飞翼布局通过中间内容包裹层的 margin 预留空间
- 面试里可补充：
  - 这类布局更多是经典面试题
  - 现代开发中通常会优先使用 Flex 或 Grid

# 追问
- 为什么这两种布局都要把中间栏写在前面？
- 双飞翼布局为什么要多包一层容器？
- 圣杯布局在缩窄时可能出现什么问题？
- 如果用 Flex 实现三栏布局，会更简单在哪里？

# 参考答案
圣杯布局和双飞翼布局，都是前端经典的三栏布局方案，目标都是一样的：左右两侧宽度固定，中间内容区域自适应，并且希望中间内容在 DOM 结构里优先出现，从而更早渲染。

圣杯布局的核心做法是：父容器先预留出左右两栏的空间，比如通过 `padding-left` 和 `padding-right`；三栏都使用浮动布局；中间栏宽度设为 100%，左右栏再通过负 `margin` 拉回两边，并结合相对定位放到预留区域里。

双飞翼布局的思路和它很像，也会让三栏浮动，中间栏先占满宽度，再通过负 `margin` 把左右栏拉到两边。不同点在于，它不会依赖父容器的 padding，而是在中间栏内部再套一层内容容器，通过这个内容容器的左右 `margin` 给两侧留出空间。

所以两者最核心的区别是“留空间的方式不同”。圣杯布局是父容器留空间，双飞翼布局是中间内容自己留空间。

双飞翼布局多包一层的好处是，中间内容区域和左右栏的关系更容易控制，在某些情况下比圣杯布局更稳定一些。圣杯布局则结构上看起来少一层，但对定位和父容器 padding 的依赖更明显。

不过从实际开发角度说，这两个更多是经典面试题。现代项目里，如果只是实现三栏布局，通常直接用 Flex 或 Grid 会更简单、可读性更高、兼容和维护成本也更低。

面试时如果你能把共同目标、各自实现思路，以及“一个靠父 padding，一个靠内部 margin”这句核心区别讲清楚，基本就够了。

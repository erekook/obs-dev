# 题目
请你讲一下前端里常见的“水平垂直居中”实现方式。你通常会怎么选择？

# 回答要点
- 常见方案不止一种，要根据场景选择。
- Flex 布局：
  - 父元素 `display: flex`
  - `justify-content: center`
  - `align-items: center`
  - 适合大多数现代布局场景
- Grid 布局：
  - `display: grid`
  - `place-items: center`
  - 写法简洁
- 绝对定位 + transform：
  - `position: absolute`
  - `top: 50%`、`left: 50%`
  - `transform: translate(-50%, -50%)`
  - 适合弹窗等脱离文档流场景
- 单行文本可用 `line-height`，但适用面窄
- 已知宽高时，也可以通过负 margin 等传统方式实现，但现代开发较少推荐
- 选择时要考虑：
  - 是否需要兼容旧浏览器
  - 子元素宽高是否固定
  - 是否脱离文档流
  - 是文本居中还是块级元素居中

# 追问
- 为什么 `margin: 0 auto` 不能直接实现垂直居中？
- 绝对定位 + transform 为什么适合未知宽高元素？
- Flex 和 Grid 做居中各有什么优缺点？
- 如果是多行文本居中，方案会有什么不同？

# 参考答案
前端里实现水平垂直居中的方法很多，没有一种方案适合所有场景。面试时最好先给出常见方案，再说明自己会如何根据场景选择。

最常用的是 Flex。只要父元素设置 `display: flex`，再加上 `justify-content: center` 和 `align-items: center`，就可以让子元素在主轴和交叉轴上同时居中。这个方案写法直观、兼容现代开发习惯，适合大多数场景。

Grid 也很方便。父元素设置 `display: grid`，再用 `place-items: center` 就能快速实现双向居中。它的优点是语法更简洁，不过在只做简单一维布局时，很多人还是更习惯用 Flex。

如果元素需要脱离文档流，比如做弹窗、悬浮层，常见方案是绝对定位配合 `transform`：先让元素 `top: 50%`、`left: 50%`，再通过 `transform: translate(-50%, -50%)` 把自身宽高的一半拉回来。这个方法的好处是即使元素宽高未知，也能正确居中。

一些特殊场景也有其他方案。比如单行文本可以通过设置等高的 `line-height` 实现垂直居中；已知宽高的块级元素也可以用绝对定位加负 margin，但这些方案适用范围更窄，现在更多是了解即可。

如果让我实际选择，我通常会这样判断：普通布局优先 Flex，网格型场景可能用 Grid，浮层弹窗这类脱离文档流元素更倾向绝对定位加 transform。面试里如果能补一句“要根据布局上下文和元素特性来选”，会显得更成熟。

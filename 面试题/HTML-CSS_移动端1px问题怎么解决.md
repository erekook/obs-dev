# 题目
请你讲一下移动端常说的“1px 问题”是什么。为什么会出现？常见解决方案有哪些？

# 回答要点
- 1px 问题本质上与设备像素比（DPR）有关。
- 在高分屏设备上，CSS 的 1px 不一定等于物理设备的 1 像素。
- 结果表现：
  - 某些边框看起来比预期粗
  - 细线不够精致
- 常见原因：
  - CSS 像素和物理像素不是一一对应
  - DPR = 2、3 等设备上，1 CSS px 会映射多个物理像素
- 常见解决方案：
  - 使用伪元素 + `transform: scale`
  - 使用 viewport 缩放方案（了解）
  - 使用 `border-image` / 渐变背景模拟细线
  - 直接使用更轻的视觉替代方案，如阴影、背景线
- 面试里可补充：
  - 不同方案的兼容性和维护成本不同
  - 现代项目里伪元素缩放方案比较常见

# 追问
- 为什么伪元素 + scale 可以实现更细的视觉边框？
- 直接写 `0.5px` 可以吗？兼容性怎么样？
- 为什么说 1px 问题本质是视觉问题而不完全是布局问题？
- 在 React/Vue 组件库里，这种细边框一般如何统一封装？

# 参考答案
移动端常说的 1px 问题，指的是我们在 CSS 里写了 `border: 1px solid`，但在高分屏手机上看起来这条线比预期更粗，不像“真正的一根物理像素线”那么细。

它出现的根本原因是 CSS 像素和设备物理像素不是一一对应的。在 DPR 为 2 或 3 的设备上，1 个 CSS 像素会映射成多个物理像素，所以你写的 `1px` 边框在视觉上会显得偏粗。

一个比较经典的解决方案是使用伪元素加 `transform: scale`。比如先让伪元素绘制一个正常边框，但把它放大到两倍尺寸，再通过 `scale(0.5)` 缩回来。这样最终视觉上就能更接近真正的“细线”效果。

另外有些环境支持直接写 `0.5px` 边框，在部分现代设备和浏览器里是可行的，但兼容性和一致性不一定完全可靠，所以很多项目仍然更倾向用伪元素方案做统一封装。

还有一些替代思路，比如使用渐变背景、`border-image`、阴影，或者干脆用背景色模拟一条分隔线。这些方案本质上都是为了得到更细、更自然的视觉效果。

所以面试里可以把核心说成两点：第一，1px 问题的根源是 DPR 导致 CSS 像素和物理像素不等价；第二，解决思路是想办法模拟更接近 1 个物理像素的视觉线条。把这两点讲清楚，通常就够了。

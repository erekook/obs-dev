# 题目
请你讲一下前端里隐藏一个元素的常见方式，比如 `display: none`、`visibility: hidden`、`opacity: 0`，它们有什么区别？

# 回答要点
- 隐藏元素不只是“看不见”，还涉及是否占位、是否可交互、是否进入可访问性树等问题。
- `display: none`
  - 元素不显示
  - 不占据布局空间
  - 不参与布局
  - 通常也不能交互
- `visibility: hidden`
  - 元素不可见
  - 仍占据原有空间
  - 仍参与布局
  - 通常不能交互
- `opacity: 0`
  - 元素完全透明
  - 仍占据空间
  - 仍参与布局
  - 默认仍可能响应点击和事件
- 其他常见方式：
  - `position` 移出可视区域
  - `clip-path` / `transform`
  - `pointer-events: none`
- 面试里可以补充：
  - 不同方式对回流、重绘、无障碍的影响不同
  - 要根据“是否保留布局、是否允许交互、是否需要动画”来选

# 追问
- 为什么 `opacity: 0` 的元素还能点击？如何避免？
- `display: none` 和 `visibility: hidden` 对屏幕阅读器有什么影响？
- 如果要做淡入淡出动画，为什么通常不会直接用 `display: none`？
- 隐藏元素时怎样兼顾性能和可访问性？

# 参考答案
前端里隐藏元素的方式很多，但它们的差别不只是“能不能看见”，而是会影响布局、交互、性能和可访问性。

`display: none` 的特点是元素彻底不参与页面布局。它既不会显示，也不会占据原来的空间，相当于从渲染树中移除了。所以切换 `display` 往往会影响布局计算，常常伴随重排。

`visibility: hidden` 则不同。元素虽然看不见，但仍然保留原来的位置和尺寸，也就是说它继续参与布局，只是视觉上隐藏了。通常它也不能被正常交互。

`opacity: 0` 更像是把元素变成完全透明。它仍然保留布局空间，也仍然在页面里，所以默认情况下依然可能响应点击、hover 和事件。这是它和前两者一个很容易被忽略的区别。如果不希望它继续交互，通常还要额外配合 `pointer-events: none`。

从使用场景看，如果你想让元素彻底消失且不占空间，通常用 `display: none`；如果只是想临时隐藏，但还保留布局位置，可以用 `visibility: hidden`；如果想做淡入淡出动画，一般更适合用 `opacity`，因为 `display` 不能直接做过渡动画。

另外，隐藏元素还可能影响无障碍体验。比如某些隐藏方式虽然视觉上不可见，但元素仍可能存在于可访问性树或焦点流程里，因此实际开发里要结合语义标签和 aria 属性一起考虑。

总结来说，这几种方式最大的区别就是：`display: none` 不占位，`visibility: hidden` 占位但不可见，`opacity: 0` 占位且默认还能交互。面试时把这三个维度说清楚，基本就够了。

# 题目
请你讲一下 CDN 的作用是什么。它大致是怎么工作的，为什么能优化前端访问速度？

# 回答要点
- CDN（Content Delivery Network）是内容分发网络。
- 核心作用：
  - 就近分发静态资源
  - 降低延迟
  - 减轻源站压力
  - 提高可用性
- 工作原理大致：
  - 用户请求资源
  - DNS / 调度系统把请求导向合适的边缘节点
  - 节点命中缓存则直接返回
  - 未命中时向源站回源拉取，再缓存并返回给用户
- 为什么更快：
  - 节点更接近用户
  - 减少跨地域网络传输
  - 边缘节点直接缓存静态资源
- 常见适用资源：
  - 图片
  - JS/CSS
  - 视频
  - 字体文件
  - 下载包
- 面试里可补充：
  - CDN 不只是加速，还可做容灾、防盗链、HTTPS 终止、压缩、边缘计算等扩展能力

# 追问
- CDN 命中和回源分别是什么意思？
- 为什么动态接口通常不能像静态资源那样简单缓存到 CDN？
- CDN 会不会带来缓存不一致问题？
- 如果资源更新了，CDN 缓存怎么刷新？

# 参考答案
CDN 的作用，简单说就是把内容提前分发到离用户更近的节点上，让用户访问资源时不必每次都跑到源站服务器去拿。

它最常见的优化对象是静态资源，比如图片、JS、CSS、字体、下载文件、视频等。这些资源通常适合缓存，一旦缓存到边缘节点，用户下次访问时就能直接从更近的节点获取。

工作过程大致是这样：用户请求一个资源时，请求不会直接固定打到源站，而是先经过 CDN 的调度系统。调度系统会根据用户所在地区、网络情况、节点健康状态等因素，把请求分配到一个合适的边缘节点。如果该节点已经缓存了这个资源，就直接返回给用户；如果没有命中缓存，就去源站拉取一份，再返回给用户，并把这份资源缓存在边缘节点里，供后续请求复用。

CDN 之所以能加速，核心原因有两个。第一，物理距离更近，网络传输路径更短；第二，很多请求不再打到源站，而是在边缘节点直接命中缓存，减少了源站的处理压力和跨地域传输耗时。

除了加速，CDN 还有很多附加价值，比如分担流量压力、提高可用性、在一定程度上增强抗攻击能力，有些 CDN 还支持图片压缩、协议优化、HTTPS 证书托管、边缘计算等能力。

不过 CDN 也不是没有问题。比如资源更新后，边缘节点可能还缓存着旧内容，所以通常要配合缓存控制、文件名 hash 或主动刷新 CDN 缓存，来解决一致性问题。

面试里如果总结一句话，可以说：CDN 本质上是把内容缓存到离用户更近的边缘节点，通过“就近访问 + 缓存命中”来降低延迟、减轻源站压力、提升访问速度。

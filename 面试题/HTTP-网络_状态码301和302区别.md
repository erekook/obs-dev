# 题目
请你讲一下 HTTP 状态码 301 和 302 的区别。它们都表示重定向，但语义和影响有什么不同？

# 回答要点
- 两者都表示重定向。
- 301：
  - 永久重定向
  - 表示资源已被永久移动到新地址
  - 浏览器和搜索引擎通常会更强地缓存和更新索引
- 302：
  - 临时重定向
  - 表示资源只是暂时跳转到新地址
  - 原地址未来仍可能恢复使用
- 常见影响：
  - SEO 处理不同
  - 浏览器缓存行为不同
  - 语义上对客户端和搜索引擎的暗示不同
- 面试里可补充：
  - 现代规范里 302/303/307 对方法保持语义更细
  - 301/302 在早期实践中对 POST 行为有历史兼容差异

# 追问
- 为什么说 SEO 更偏向把权重传递给 301 的新地址？
- 307 和 302 有什么区别？
- 为什么某些跳转场景不适合用 301？
- 浏览器对 301 缓存后可能带来什么调试麻烦？

# 参考答案
301 和 302 都表示服务器告诉客户端：“你请求的资源现在应该去另一个地址找。”但它们最大的区别在于，这个跳转是永久的，还是临时的。

301 表示永久重定向，也就是资源已经被永久迁移到新的 URL。浏览器、搜索引擎和各种客户端通常会把这个信号理解成“以后就优先用新地址了”。因此 301 往往会有更明显的缓存效果，搜索引擎也更倾向把旧地址的权重逐步转移到新地址。

302 表示临时重定向，意思是当前先跳到新地址，但原始地址未来仍可能恢复使用。它更适合短期跳转、活动页切换、登录态跳转、流量分流等场景。

从实际影响上看，301 和 302 不只是状态码不同，它们是在向浏览器和搜索引擎表达不同的语义。也正因为这样，如果一个跳转只是临时策略，却错误地用了 301，浏览器可能把它缓存很久，后面你即使改回去，客户端也可能还一直往旧的重定向规则走，调试时会很麻烦。

另外，面试里有时会延伸到方法保持的问题。历史上浏览器对 301 和 302 的处理有一些兼容行为，比如 POST 请求可能被改成 GET。现代规范里为了更明确地区分这种语义，又引入了 307、308 等状态码。

如果面试里总结，可以说：301 是永久重定向，302 是临时重定向；它们的核心区别不只是“缓存时间不同”，而是对客户端和搜索引擎表达的迁移语义不同。

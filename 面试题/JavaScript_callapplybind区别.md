# 题目
请你讲一下 JavaScript 里 `call`、`apply`、`bind` 的区别。它们分别适合什么场景？

# 回答要点
- 三者都能显式指定函数执行时的 `this`。
- `call`：
  - 立即调用函数
  - 参数逐个传入
- `apply`：
  - 立即调用函数
  - 参数以数组或类数组形式传入
- `bind`：
  - 不会立即执行
  - 返回一个绑定了 `this` 的新函数
  - 可用于偏函数预置参数
- 常见场景：
  - `call` / `apply`：临时借用函数、改变执行上下文
  - `apply`：已有参数数组时调用更方便
  - `bind`：事件回调、延迟执行、预绑定 `this`
- 面试里可补充：
  - `bind` 返回的新函数可继续接收参数
  - `new` 调用绑定函数时有优先级问题

# 追问
- 为什么 `bind` 不会立刻执行？
- `call` 和 `apply` 在现代开发里差异还大吗？
- `bind` 的偏函数能力怎么理解？
- 如果把 `bind` 返回的函数拿去 `new`，`this` 指向谁？

# 参考答案
`call`、`apply`、`bind` 的共同点，是它们都能显式指定函数执行时的 `this`。区别主要在于“是否立即执行”和“参数怎么传”。

`call` 会立即执行函数，并且参数是一个一个传进去的，比如：

```js
fn.call(obj, a, b, c)
```

这里的意思就是让 `fn` 以 `obj` 作为 `this` 立即执行。

`apply` 和 `call` 很像，也会立即执行，只不过参数通常要放在数组或类数组里：

```js
fn.apply(obj, [a, b, c])
```

所以如果你手里本来就是一个参数数组，`apply` 会更顺手。

`bind` 则不一样。它不会立刻调用函数，而是返回一个新的函数，这个新函数的 `this` 已经被提前绑定好了。

```js
const newFn = fn.bind(obj)
```

之后什么时候执行 `newFn`，由你自己决定。也正因为它返回的是新函数，所以特别适合做事件回调、定时器回调，或者需要提前固定 `this` 的场景。

另外，`bind` 还可以顺手预置一部分参数，这就是偏函数的一种常见用法。

如果从使用场景来总结，`call` 和 `apply` 更适合“马上换个 this 执行一下”，而 `bind` 更适合“先生成一个 this 已经固定好的新函数，留着以后再执行”。

面试里还可以补一句：在现代开发里，`call` 和 `apply` 的核心差别其实主要就是参数形式，而 `bind` 最大的不同是它不会立即执行。把这三点讲清楚，通常就够了。

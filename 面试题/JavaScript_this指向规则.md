# 题目
请你讲一下 JavaScript 里 `this` 的指向规则。不同调用方式下，`this` 分别指向什么？

# 回答要点
- `this` 不是定义时确定，而是大多数情况下在调用时确定。
- 常见绑定规则：
  - 默认绑定
  - 隐式绑定
  - 显式绑定（`call` / `apply` / `bind`）
  - `new` 绑定
- 默认绑定：
  - 普通函数直接调用
  - 非严格模式下浏览器通常指向 `window`
  - 严格模式下为 `undefined`
- 隐式绑定：
  - 通过对象调用方法时，`this` 指向调用它的对象
- 显式绑定：
  - `call`、`apply`、`bind` 可手动指定 `this`
- `new` 绑定：
  - `this` 指向新创建的实例对象
- 箭头函数特殊点：
  - 没有自己的 `this`
  - `this` 取决于外层词法作用域
- 面试里可补充：
  - `bind` 返回新函数
  - 箭头函数不能作为构造函数使用

# 追问
- `call`、`apply`、`bind` 的区别是什么？
- 为什么把对象方法赋值给变量后再调用，`this` 可能变了？
- 箭头函数为什么不适合定义对象方法或构造函数？
- `new` 绑定和 `bind` 绑定一起出现时谁优先？

# 参考答案
JavaScript 里的 `this` 是一个非常高频的面试题，核心点在于：`this` 大多数时候不是在函数定义时决定的，而是在函数执行时，根据调用方式决定的。

第一种是默认绑定。也就是普通函数直接调用，没有明显调用者时，非严格模式下浏览器里通常会指向全局对象 `window`，严格模式下则是 `undefined`。

第二种是隐式绑定。当函数作为对象的方法被调用时，`this` 通常指向调用它的那个对象。比如 `obj.say()`，这时候 `say` 里的 `this` 一般就是 `obj`。

第三种是显式绑定。通过 `call`、`apply`、`bind`，可以手动指定函数执行时的 `this`。其中 `call` 和 `apply` 会立即调用函数，只是传参形式不同；`bind` 不会立刻执行，而是返回一个绑定好 `this` 的新函数。

第四种是 `new` 绑定。当一个函数通过 `new` 调用时，会创建一个新的对象，并让函数体内的 `this` 指向这个新对象，所以它常用于构造函数场景。

箭头函数是一个特殊点。它没有自己的 `this`，不会按照上面这些普通规则重新绑定，而是直接捕获外层作用域的 `this`。所以箭头函数很适合回调场景，但不适合作为构造函数，也不适合随便拿来写需要动态 `this` 的对象方法。

如果要总结优先级，通常可以理解为：`new` 绑定优先级很高，显式绑定次之，隐式绑定再次，最后才是默认绑定。面试里把这些调用方式分开讲，再配合箭头函数的特殊性，基本就很完整了。

# 题目
请你讲一下 JavaScript 里 `var`、`let`、`const` 的区别。除了作用域，还有哪些常见面试点？

# 回答要点
- 三者都用于声明变量，但行为有明显差异。
- `var`：
  - 函数作用域
  - 存在变量提升
  - 可重复声明
  - 全局声明时可能挂到 `window`
- `let`：
  - 块级作用域
  - 存在暂时性死区（TDZ）
  - 不可重复声明
- `const`：
  - 块级作用域
  - 同样有暂时性死区
  - 声明时必须初始化
  - 绑定不可重新赋值，但引用类型内部值可变
- 常见面试点：
  - 变量提升区别
  - for 循环闭包问题
  - 全局对象挂载差异
  - `const` 并不等于值完全不可变
- 面试里可补充：
  - 默认优先 `const`，需要重新赋值时再用 `let`
  - `var` 在现代开发中应尽量少用

# 追问
- 什么是暂时性死区？
- 为什么 `const` 声明对象后还能改属性？
- `let` 如何解决 `var` 在循环闭包中的经典问题？
- 为什么全局 `let/const` 不会像 `var` 那样成为 `window` 的属性？

# 参考答案
`var`、`let`、`const` 都能用来声明变量，但它们在作用域、提升行为、重复声明、赋值规则上都有差异。

先说 `var`。它是函数作用域，不是块级作用域，所以在 `if`、`for` 这种代码块里声明，出了代码块依然可能访问到。另外它存在变量提升，也就是说声明会被提升到当前作用域顶部，只是值不会一起提升，所以在声明前访问通常得到 `undefined`。而且 `var` 还能重复声明，在全局环境里声明时还可能挂到 `window` 上。

`let` 和 `const` 是 ES6 引入的块级作用域声明方式。它们都只在当前代码块内有效，不会像 `var` 那样轻易泄漏到外部。它们也有提升，但不像 `var` 那样可以直接访问，因为在声明前会处于“暂时性死区”，这时候访问会报错。

`let` 适合需要重新赋值的变量，而 `const` 适合声明后不希望再重新绑定的变量。`const` 声明时必须初始化，这是它和 `let` 的一个明显区别。

不过 `const` 常被误解成“完全不可变”。其实它只是变量绑定不能重新指向别的值。如果它保存的是对象或数组，内部属性和元素仍然可以修改。

现代开发里，一般推荐默认优先使用 `const`，只有确实需要重新赋值时再用 `let`，而 `var` 尽量少用。因为 `let/const` 在作用域和语义上都更清晰，也能避免很多历史遗留问题。

如果面试里总结，可以说：`var` 是函数作用域且可重复声明，`let` 是块级作用域可重新赋值，`const` 是块级作用域且绑定不可重新赋值；再补上变量提升、暂时性死区和 `const` 的引用类型特点，就比较完整了。

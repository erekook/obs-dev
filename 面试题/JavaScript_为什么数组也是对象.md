# 题目
为什么在 JavaScript 里，数组本质上也被说成是对象？它和普通对象有什么相同点和不同点？

# 回答要点
- 在 JavaScript 中，数组属于引用类型，本质上是一种特殊对象。
- 相同点：
  - 都可以存属性
  - 都通过引用来比较
  - `typeof` 对数组和普通对象都返回 `object`
- 不同点：
  - 数组是为有序数据设计的特殊结构
  - 有 `length` 属性
  - 有一套专门的数组方法（`push`、`pop`、`map` 等）
  - 数字索引有特殊语义和行为
- 面试里可补充：
  - 判断数组不能只用 `typeof`
  - 常见正确方式是 `Array.isArray()`

# 追问
- 为什么 `typeof []` 会是 `object`？
- 数组索引和对象键在底层理解上有什么关系？
- 为什么数组更适合顺序数据而对象更适合键值映射？
- 判断数组还有哪些方式，为什么 `Array.isArray` 更推荐？

# 参考答案
说数组本质上也是对象，核心是因为在 JavaScript 里，数组并不是一种完全独立于对象体系之外的值类型，它属于对象类型的一种特殊形式。

这点最直接的体现就是：

```js
typeof [] // 'object'
```

数组和普通对象有很多共同点。它们都属于引用类型，赋值和比较时传递的是引用；都可以动态添加属性；都能挂方法；也都继承自对象体系。

但数组和普通对象的用途并不一样。数组是专门为“有顺序的数据集合”设计的，所以它有 `length` 属性，有 `push`、`pop`、`map`、`filter` 这类专门的方法，也会把数字索引当成一类特殊键来处理。

普通对象则更适合表示无序的键值映射，比如用户信息、配置对象、字典结构等。虽然你也能给对象写数字键，但它不会天然像数组那样帮你维护一整套顺序结构和长度语义。

所以可以把数组理解成：“站在对象系统里的一个特化版本”，它依然是对象，但为了顺序数据操作做了额外封装和优化。

这也是为什么判断一个值是不是数组，不能只靠 `typeof`。因为数组和普通对象都会返回 `object`。更推荐的方式是：

```js
Array.isArray(value)
```

面试里如果总结，可以说：数组在 JavaScript 里本质上是特殊对象，共享对象的很多特征，但额外提供了顺序数据结构的语义、长度管理和专用方法，所以“它是对象”没错，但“它只是普通对象”也不准确。

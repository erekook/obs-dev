# 题目
为什么说 JavaScript 是基于原型，而不是基于类的？

# 你的回答
JS 是通过原型来实现继承的，ES6 中有 class 关键字，但是背后实际原理还是原型实现的。

# 面试点评
答到点上了。可以再补一句“传统面向对象语言是基于类创建实例，而 JS 更核心的是对象之间通过原型关联”。

# 回答要点
- JS 的继承机制本质是通过原型链实现的
- 对象和对象之间通过原型关联
- 属性和方法查找依赖原型链
- ES6 class 只是语法糖
- 底层依然是原型和原型链

# 参考答案
JavaScript 之所以说是基于原型，而不是基于类，是因为它的继承机制本质上是通过对象之间的原型链来实现的。一个对象可以通过原型关联到另一个对象，属性和方法的查找也是沿着原型链进行的。虽然 ES6 提供了 `class` 关键字，但它本质上只是语法糖，底层实现依然是基于原型和原型链。

# 题目
请你讲一下 JavaScript 里的原型（prototype）和原型链是什么。它们之间是什么关系？

# 回答要点
- JavaScript 采用基于原型的对象机制。
- 原型相关几个核心概念：
  - 构造函数的 `prototype`
  - 实例对象的隐式原型 `[[Prototype]]`（常见访问形式 `__proto__`）
  - 原型链查找
- 关系：
  - 构造函数的 `prototype` 是实例共享成员的地方
  - 实例对象的隐式原型通常指向构造函数的 `prototype`
- 原型链：
  - 对象访问属性时先找自身
  - 找不到就沿着原型向上查找
  - 一直到 `null` 为止
- 面试里可补充：
  - `class` 本质上仍建立在原型之上
  - `prototype` 是函数显式属性，`__proto__` 更像对象与原型之间的链接

# 追问
- `prototype` 和 `__proto__` 有什么区别？
- 为什么把方法挂在 `prototype` 上能节省内存？
- `Object.prototype` 的原型最终为什么是 `null`？
- 手写 `new` 时为什么要把实例的原型指向构造函数的 `prototype`？

# 参考答案
JavaScript 里的原型和原型链，本质上是在解决“对象如何复用属性和方法”的问题。

先说原型。每个构造函数都有一个 `prototype` 属性，这个属性指向一个对象，通常我们把它叫做“原型对象”。如果一个对象是通过这个构造函数创建出来的，那么这个对象通常会通过自己的隐式原型链接到这个原型对象上。

比如：

```js
function Person(name) {
  this.name = name;
}
Person.prototype.sayHi = function () {
  console.log('Hi');
};

const p = new Person('Tom');
```

这里 `Person.prototype` 上的 `sayHi` 方法不会为每个实例都单独复制一份，而是所有实例共享这一份方法，这样更节省内存。

再说原型链。对象访问一个属性时，浏览器或运行时会先看对象自身有没有。如果没有，就顺着这个对象的隐式原型往上找，也就是去它的原型对象上找；如果还没有，就继续往更上一层原型找，一直找到 `null` 为止。这个一层层向上查找的过程，就是原型链。

所以可以简单理解为：原型是“共享属性和方法的对象”，原型链是“查找属性时沿着原型逐级向上找的路径”。

很多人容易混淆 `prototype` 和 `__proto__`。可以这么记：`prototype` 是构造函数上的属性，用来给实例提供共享成员；`__proto__` 更像实例对象内部指向原型对象的那条链接。

现代 JavaScript 里的 `class` 看起来像类继承，但底层本质上仍然是原型链机制在工作。

面试里如果总结，可以说：原型是对象共享成员的基础，原型链是对象查找属性的规则；前者决定“共享在哪里”，后者决定“查找怎么走”。

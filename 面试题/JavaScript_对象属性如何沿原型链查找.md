# 题目
对象在查找一个属性时，具体是怎么沿原型链查找的？

# 你的回答
先找该对象自身的属性，如果没找到会往上找创建该对象的构造函数的 prototype 上找这个属性，如果还没有找到，会继续往上找，直到找到 Object 的 prototype，最后找到 null，返回 undefined。

# 面试点评
回答已经比较完整。更准确的说法是“沿着对象的原型链向上查找”，因为核心是原型链本身，而不是一定要从构造函数角度描述。还可以补充一句“只要某一层找到就立即停止查找”。

# 回答要点
- 先查找对象自身属性
- 没有就沿原型链向上找
- 原型对象本身也可能有原型
- 会一直查找到 Object.prototype
- 再往上是 null
- 没找到返回 undefined
- 查找命中就停止

# 参考答案
访问对象属性时，浏览器会先查找对象自身是否有这个属性。如果没有，就会沿着对象的原型链向上查找，也就是到它的原型对象上找。如果原型对象上还没有，就继续沿原型链往上，一直找到 `Object.prototype`。如果在 `Object.prototype` 上也没有，再继续往上会到 `null`，此时查找结束，返回 `undefined`。在查找过程中，只要某一层找到该属性，就会立即停止继续向上查找。

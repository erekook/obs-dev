# 题目
JavaScript 里数组去重有哪些常见方式？如果让你选一种最推荐的写法，你会怎么说？

# 回答要点
- 常见去重方式：
  - `Set`
  - `filter + indexOf`
  - `reduce`
  - 对象/Map 记录出现过的值
- 最常见推荐：
  - `Array.from(new Set(arr))`
  - 或 `[...new Set(arr)]`
- 各方式特点：
  - `Set` 简洁、语义清晰、性能通常较好
  - `filter + indexOf` 直观，但效率相对一般
  - 对象/Map 方案适合定制键或复杂场景
- 面试里可补充：
  - 基本类型去重和对象去重要区分
  - `Set` 对对象是按引用去重，不是按内容去重

# 追问
- 为什么两个内容一样的对象放进 `Set` 里还是不会被去重？
- 如果数组元素是对象，按某个字段去重该怎么做？
- `Set` 去重背后依赖的是哪种相等判断语义？
- 为什么说去重策略要看“值类型”还是“业务键”？

# 参考答案
JavaScript 数组去重最常见的方式有好几种，但如果是普通面试题，最推荐也最常用的通常是 `Set`。

比如：

```js
const result = [...new Set(arr)];
```

或者：

```js
const result = Array.from(new Set(arr));
```

这种写法简洁、语义明确，而且对基本类型数组去重非常方便。

除了 `Set`，还有一些经典写法，比如：

```js
arr.filter((item, index) => arr.indexOf(item) === index)
```

这也能实现去重，但本质上需要反复查找，数据量大时性能通常不如 `Set`。

还可以用 `reduce` 或对象/Map 记录是否见过某个值，这些方式更灵活，适合自定义规则场景。

不过面试里有一个重要补充点：如果数组元素是对象，`Set` 并不会按“对象内容相同”来去重，而是按“引用是否相同”来判断。也就是说，两个长得一模一样但不是同一个引用的对象，放进 `Set` 后仍然会保留两份。

所以如果是对象数组去重，通常要按某个业务字段来做，比如 id。这个时候更常见的写法是用 `Map` 或对象记录已经出现过的 key。

比如：

```js
const result = [];
const seen = new Set();
for (const item of arr) {
  if (!seen.has(item.id)) {
    seen.add(item.id);
    result.push(item);
  }
}
```

面试里如果总结，可以说：普通数组去重优先推荐 `Set`，简单高效；如果是对象数组，关键要先明确“按引用去重”还是“按字段去重”，再选合适方案。

# 题目
请你讲一下 JavaScript 里箭头函数和普通函数有什么区别。除了写法简洁之外，还有哪些本质差异？

# 回答要点
- 箭头函数不仅是语法糖，还有行为差异。
- 主要区别：
  - 没有自己的 `this`
  - 没有自己的 `arguments`
  - 不能作为构造函数，不能用 `new`
  - 没有 `prototype`
  - 更适合简短回调和保留外层上下文
- `this` 差异：
  - 普通函数的 `this` 取决于调用方式
  - 箭头函数的 `this` 取决于定义时外层词法作用域
- 常见场景：
  - 箭头函数适合回调、数组方法、需要继承外层 `this` 的场景
  - 普通函数适合作为对象方法、构造函数、需要动态 `this` 的函数
- 面试里可补充：
  - 不是所有函数都应该写成箭头函数
  - 对象方法里滥用箭头函数是常见坑

# 追问
- 为什么箭头函数不适合做对象方法？
- 没有 `arguments` 时如何拿到参数列表？
- 为什么箭头函数不能被 `new` 调用？
- class 方法里什么时候用箭头函数，什么时候用普通方法？

# 参考答案
箭头函数和普通函数的区别，绝不只是“写法更短”这么简单，它们在 `this`、构造能力和函数内部行为上都有本质差异。

最重要的一点是 `this`。普通函数的 `this` 取决于调用方式，比如对象调用、显式绑定、`new` 调用等；而箭头函数没有自己的 `this`，它会直接捕获定义时所在外层作用域的 `this`。这也是为什么箭头函数特别适合写回调，因为它能避免传统回调里 `this` 丢失的问题。

第二个区别是 `arguments`。普通函数有自己的 `arguments` 对象，而箭头函数没有，它会去外层作用域里找。如果需要更现代的参数收集方式，通常会直接用剩余参数 `...args`。

第三，箭头函数不能当构造函数使用，也就是说不能用 `new` 去调用。因为它没有自己的 `this` 绑定逻辑，也没有 `prototype`，所以不适合拿来创建实例对象。

这也延伸出一个常见面试点：箭头函数通常不适合作为对象方法。因为对象方法往往希望 `this` 指向当前对象，而箭头函数不会根据调用者动态改变 `this`，很容易拿到外层上下文，导致行为不符合预期。

所以实际使用时，可以这样理解：箭头函数适合简短回调、数组方法、Promise 回调、需要继承外层 `this` 的场景；普通函数更适合作为对象方法、构造函数，或者任何需要动态 `this` 的地方。

面试里如果总结，可以说：箭头函数的核心差异不是“短”，而是它没有自己的 `this`、`arguments`、`prototype`，也不能被 `new`；因此它更适合回调，不适合所有函数场景。

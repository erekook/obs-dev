# 题目
请你讲一下什么是闭包。闭包的本质是什么？它有哪些常见应用场景和潜在问题？

# 回答要点
- 闭包不是单独语法，而是一种现象和能力。
- 本质：函数可以访问其词法作用域中的变量，即使在外部作用域已经执行结束后仍然如此。
- 形成条件：
  - 函数嵌套
  - 内部函数引用了外部函数的变量
  - 内部函数在外部作用域之外仍被使用
- 常见应用：
  - 封装私有变量
  - 柯里化
  - 防抖/节流
  - 函数工厂
  - 回调中保留上下文数据
- 潜在问题：
  - 变量长期不能释放，可能造成内存占用增加
  - 循环里使用不当可能出现取值错误
- 面试里可补充：
  - 闭包的关键是“变量被引用而存活”
  - 不要只背“函数套函数”

# 追问
- 为什么说“函数套函数”不一定就是闭包？
- 闭包为什么能让外部函数执行结束后变量还存在？
- `let` 和 `var` 在循环配合闭包时有什么差异？
- 如何避免闭包造成不必要的内存占用？

# 参考答案
闭包是 JavaScript 里非常经典的概念，但它不是某个单独的语法，而是一种运行现象。最常见的理解是：内部函数可以访问外部函数作用域里的变量，并且即使外部函数已经执行结束，这些变量仍然不会马上销毁。

比如下面这个例子：

```js
function outer() {
  let count = 0;
  return function () {
    count++;
    return count;
  };
}

const fn = outer();
fn(); // 1
fn(); // 2
```

这里 `outer` 执行结束后，本来它的局部变量 `count` 应该被销毁，但因为返回出去的内部函数还在引用它，所以这个变量会继续存活。这个现象就是闭包的核心。

所以闭包的本质不是“函数里又写了一个函数”，而是“函数访问了其词法作用域中的变量，并让这些变量在原作用域结束后仍然可用”。

闭包有很多实际用途。比如封装私有变量、实现函数工厂、做柯里化、防抖节流、在异步回调里保留上下文数据等。很多常见工具函数底层其实都在利用闭包。

当然，闭包也有代价。因为被引用的变量不会及时释放，如果使用不当，可能导致不必要的内存占用。另外在循环里如果配合 `var` 使用不当，也容易出现经典的取值错误问题。

面试时比较加分的一点是，不要把闭包只回答成“函数套函数”。更准确的说法应该是：闭包让函数能够记住并持续访问其定义时所在作用域中的变量。把这句话讲清楚，通常就够了。

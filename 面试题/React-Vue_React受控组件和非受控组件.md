# 题目
请你讲一下 React 里的受控组件和非受控组件。它们分别是什么意思，适合什么场景？

# 回答要点
- 受控组件：
  - 表单值由 React state 管理
  - 输入值通过 `value` 绑定
  - 通过 `onChange` 更新 state
- 非受控组件：
  - 表单值由 DOM 自己维护
  - 通常通过 `ref` 获取当前值
  - 常使用 `defaultValue`
- 受控组件优点：
  - 数据流清晰
  - 易于做实时校验、联动、条件渲染
  - 更符合 React 声明式思想
- 非受控组件优点：
  - 写法简单
  - 某些场景性能或接入第三方库更方便
- 适用场景：
  - 复杂表单、实时校验更适合受控组件
  - 简单表单、文件输入、快速接入原生或第三方 DOM 逻辑时可考虑非受控组件
- 面试里可补充：
  - 文件上传 input 通常是非受控的典型场景
  - 受控和非受控不要混用得太混乱

# 追问
- 为什么说受控组件更符合 React 的单向数据流？
- 文件输入为什么通常是非受控组件？
- 大型表单里全受控会有什么代价？
- React Hook Form 这类库为什么常结合非受控思路做优化？

# 参考答案
React 里的受控组件和非受控组件，主要区别在于“表单值由谁管理”。

受控组件指的是，表单输入值不直接交给 DOM 自己维护，而是完全交给 React 的 state 管理。组件通过 `value` 把 state 的值绑定到输入框上，再通过 `onChange` 把用户输入同步回 state。也就是说，输入框里显示什么，最终由 React 决定。

这种方式的好处是数据流非常清晰。因为表单值都在 React 状态里，所以做实时校验、输入联动、条件渲染、统一提交处理都比较方便，也更符合 React 的声明式思想。

非受控组件则相反。它把表单值交给 DOM 自己维护，React 不去持续接管这个值。需要读取时，再通过 `ref` 从 DOM 上拿当前值。它常用 `defaultValue` 而不是 `value`。

非受控组件的优点是写法更轻，有些场景下性能更省，也更方便接入一些偏原生 DOM 的逻辑。比如文件上传的 `input[type=file]`，通常就是典型的非受控组件场景，因为文件值本身并不是普通字符串状态那样由 React 完整控制。

实际开发里，如果表单逻辑复杂，比如需要实时校验、联动显示、受条件影响禁用启用等，通常更适合受控组件。如果表单很简单，或者只是在提交时取一次值，非受控组件也可以。

面试里可以总结成一句话：受控组件是“React 管值”，非受控组件是“DOM 管值”；前者更清晰更适合复杂交互，后者更轻量更适合简单场景或特殊输入。

# 题目
请你讲一下 Vue 的响应式原理。Vue2 和 Vue3 在实现响应式方面有什么核心区别？

# 回答要点
- 响应式的目标是：数据变化后，视图能自动更新。
- Vue2 的核心实现：
  - `Object.defineProperty`
  - 初始化时递归劫持对象属性
  - 依赖收集与派发更新
- Vue2 的局限：
  - 不能直接监听对象新增/删除属性
  - 不能天然监听数组索引和 length 变化
  - 深层对象初始化成本较高
- Vue3 的核心实现：
  - `Proxy`
  - 可以拦截更多操作，如 `get`、`set`、`deleteProperty`
  - 响应式更完整、按需代理更灵活
- 共同点：
  - 都有依赖收集
  - 都会在数据变更时触发副作用更新
- 面试回答里可提到：
  - `track` / `trigger`
  - `effect`
  - `computed`、`watch` 本质上建立在响应式系统之上

# 追问
- 为什么 Vue2 需要 `Vue.set`？
- `Proxy` 为什么被认为比 `Object.defineProperty` 更适合响应式？
- `computed` 和 `watch` 在底层依赖响应式系统时有什么不同？
- 为什么 Vue3 的响应式对 Map、Set 支持更自然？

# 参考答案
Vue 的响应式原理，本质上是在数据访问和修改时做拦截。读取数据时，框架会记录“谁依赖了这个数据”；修改数据时，再通知相关依赖重新执行，从而驱动视图更新。

Vue2 主要基于 `Object.defineProperty` 实现。它会在初始化阶段递归遍历 `data` 中的属性，把每个属性转换成带有 `getter` 和 `setter` 的形式。这样当读取属性时，可以做依赖收集；当设置属性时，可以触发更新。

Vue2 的问题是，`Object.defineProperty` 只能劫持已经存在的属性，所以对象新增和删除属性默认无法被监听，需要借助 `Vue.set` 或 `Vue.delete`。数组这边也有局限，比如直接通过下标修改或修改 `length`，不能被完整感知，所以 Vue2 额外重写了部分数组方法。

Vue3 改用 `Proxy` 实现响应式。`Proxy` 是对整个对象做代理，不是逐个属性定义 getter/setter，因此它可以更自然地拦截属性新增、删除、遍历，以及数组、Map、Set 等结构的变化。整体上能力更完整，也减少了 Vue2 那种初始化阶段深度递归劫持的限制。

从更底层的角度看，Vue3 在读取时会 `track` 依赖，在修改时会 `trigger` 更新，副作用函数通常通过 `effect` 建立关联。`computed` 和 `watch` 都是建立在这套响应式系统之上的高级能力。

如果用一句话总结：Vue2 用 `Object.defineProperty` 做“属性级劫持”，Vue3 用 `Proxy` 做“对象级代理”，后者在能力、灵活性和对复杂数据结构的支持上都更强。

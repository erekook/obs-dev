# 题目
请你讲一下 Vue 里的 `computed` 和 `watch` 有什么区别。它们分别适合什么场景？

# 回答要点
- 两者都和响应式数据变化有关，但用途不同。
- `computed`：
  - 用于基于已有状态派生出新值
  - 有缓存
  - 依赖不变时不会重复计算
  - 更适合模板展示和纯计算逻辑
- `watch`：
  - 用于监听数据变化并执行副作用
  - 常用于异步请求、日志、手动同步其他状态
  - 没有“派生值缓存”这个核心特点
- 选择原则：
  - 能用 `computed` 解决的，不要用 `watch` 硬算
  - 需要副作用时再用 `watch`
- 面试里可补充：
  - `computed` 更声明式
  - `watch` 更偏命令式
  - `watch` 可监听多个源、深度监听、立即执行等

# 追问
- 为什么说很多 `watch` 其实可以改写成 `computed`？
- `computed` 为什么有缓存？
- 深度监听 `watch` 为什么可能带来性能问题？
- Vue3 里的 `watchEffect` 和 `watch` 又有什么区别？

# 参考答案
`computed` 和 `watch` 都建立在 Vue 的响应式系统之上，但职责并不一样。

`computed` 的核心作用是“根据已有响应式数据，计算出一个新的派生值”。比如根据商品单价和数量计算总价，根据多个字段拼接展示文案，这类场景都很适合用 `computed`。它最大的特点是有缓存，只要它依赖的数据没有变化，就不会重复执行计算逻辑。

`watch` 的核心作用是“监听某个数据变化，然后执行一段副作用逻辑”。比如数据变化后发请求、写日志、同步本地存储、手动更新其他状态，这类操作更适合用 `watch`。

所以一个简单判断标准是：如果你只是想得到一个新值，优先考虑 `computed`；如果你需要在变化发生后做一件事，尤其是带副作用的事，才用 `watch`。

很多初学者会把 `watch` 当成万能工具，比如监听 `firstName` 和 `lastName`，然后手动更新一个 `fullName`。这种写法其实就不如直接用 `computed` 来得自然，因为 `fullName` 本质上只是派生结果，不需要额外手动维护。

`computed` 更像声明式地描述“这个值由哪些数据算出来”，而 `watch` 更像命令式地说“当这个值变了，你去做点什么”。这也是它们在代码可维护性上的重要区别。

另外，`watch` 还支持深度监听、立即执行、监听多个数据源等功能，但这些能力也意味着如果使用不当，可能带来额外性能开销。

面试里如果总结成一句话，可以说：`computed` 用来求值，`watch` 用来做事；前者强调派生和缓存，后者强调监听和副作用。

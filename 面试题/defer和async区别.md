# 题目
defer 和 async 的区别是什么？

# 你的回答
defer 是不会阻塞 HTML 解析，等 DOM 解析完后执行；async 是下载完立刻执行。

# 面试点评
方向正确，已经说出了执行时机的核心区别。
但如果要拿更稳的面试分，还需要补上两个关键点：
1. defer 和 async 都是“下载不阻塞”
2. defer 有执行顺序，async 没有执行顺序保证

# 回答要点
- defer 和 async 都会并行下载脚本
- defer 在 HTML 解析完成后执行
- defer 会按照脚本出现顺序依次执行
- async 下载完成就立即执行
- async 执行顺序不保证
- async 可能在 HTML 尚未解析完成时打断解析

# 更完整的参考答案
`defer` 和 `async` 都不会阻塞脚本文件的下载，它们都会并行下载。
区别在于执行时机不同。
`defer` 会等 HTML 解析完成后，再按脚本在页面中的顺序依次执行。
`async` 是脚本下载完成后立即执行，执行顺序不保证，如果这时 HTML 还没解析完，可能会打断解析。
所以可以简单记成：
- defer：延迟执行，有顺序，适合多个有依赖关系的脚本
- async：下载完马上执行，无顺序，适合独立脚本，比如埋点、统计代码

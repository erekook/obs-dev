# 题目
为什么普通 script 会阻塞 HTML 解析？

# 你的回答
因为 script 可能会操作 DOM。

# 面试点评
这个回答抓住了核心，但太短。面试里需要把“为什么浏览器要暂停解析”讲清楚。
重点不是只说“会操作 DOM”，而是要说明浏览器为了保证 DOM 构建结果正确，必须暂停 HTML 解析，先执行脚本，再继续往后解析。

# 回答要点
- 浏览器解析 HTML 时遇到普通 script，会暂停 HTML 解析
- 因为脚本可能读取或修改当前 DOM 结构
- 为保证 DOM 构建结果正确，必须先执行 JS 再继续解析
- 例如 document.write、增删节点、修改属性等都会影响后续 DOM
- 普通 script 会阻塞 DOM 解析
- defer 和 async 不同于普通 script

# 更完整的参考答案
因为浏览器在解析 HTML 时，如果遇到普通 script 标签，不知道这段 JS 会不会直接修改当前正在解析的 DOM 结构。
为了保证 DOM 构建结果正确，浏览器会先暂停 HTML 解析，下载并执行这段 JS，等执行完成后再继续解析后面的 HTML。
如果脚本里调用了 document.write，或者执行了增删节点、修改属性等操作，就会影响后续 DOM 结构，所以浏览器不能一边不受控制地继续解析，一边又执行脚本。
这也是为什么普通 script 会阻塞 DOM 解析。

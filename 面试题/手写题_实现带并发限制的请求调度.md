# 题目
手写一个带并发限制的请求调度器：有一批异步任务，但同一时间最多只能并发执行 `n` 个，你会怎么实现？

# 回答要点
- 核心目标：
  - 控制同一时刻运行中的任务数不超过上限
- 基本思路：
  - 维护任务队列
  - 维护当前运行中的数量
  - 每完成一个任务，就补一个新的任务进来
- 关键点：
  - 任务完成或失败都要释放并发名额
  - 最终结果顺序通常要和输入顺序一致
  - 全部任务结束后返回整体结果
- 面试里可补充：
  - 这是典型的“队列 + 计数器”问题
  - 实际场景可用于批量请求、上传、爬取、资源预加载等

# 追问
- 为什么不能只用 `Promise.all`？
- 如果某个任务失败了，是继续调度还是整体中断？
- 怎么保证返回结果顺序和输入顺序一致？
- 这个实现能不能扩展成动态添加任务？

# 参考答案
这个题的关键不是“会不会写 Promise”，而是能不能把“任务队列”和“并发名额”两个概念理清楚。

思路通常是：
1. 先准备一个任务数组，每个任务都是返回 Promise 的函数；
2. 维护一个 `running`，表示当前正在执行的任务数；
3. 每次只要 `running < limit`，就从队列里取一个任务执行；
4. 某个任务完成或失败后，把 `running` 减一，再继续补下一个任务；
5. 当所有任务都处理完后，返回结果。

一个常见实现如下：

```js
function runWithLimit(tasks, limit) {
  return new Promise((resolve, reject) => {
    const results = new Array(tasks.length);
    let nextIndex = 0;
    let finished = 0;
    let running = 0;

    function runNext() {
      if (finished === tasks.length) {
        resolve(results);
        return;
      }

      while (running < limit && nextIndex < tasks.length) {
        const current = nextIndex;
        const task = tasks[current];
        nextIndex++;
        running++;

        Promise.resolve()
          .then(() => task())
          .then((res) => {
            results[current] = res;
          })
          .catch((err) => {
            results[current] = err;
          })
          .finally(() => {
            running--;
            finished++;
            runNext();
          });
      }
    }

    runNext();
  });
}
```

如果希望“某个任务失败就整体 reject”，也可以在 `catch` 里直接 `reject(err)`，并停止后续调度。这里要看题目要求。

这个题的加分点通常有两个：
- 一是能考虑失败分支也要释放并发名额；
- 二是能考虑结果顺序要按原任务顺序保存，而不是按完成顺序返回。

面试里如果总结，可以说：并发限制调度的本质，是用队列管理待执行任务，用计数器控制运行中的任务数，每当一个任务结束就补上新的任务，直到全部完成。

# 题目
请手写一个 `debounce` 函数，并说明它的应用场景。进一步说一下，防抖和节流（throttle）的区别是什么？

# 回答要点
- 防抖含义：高频触发事件时，只在最后一次触发后的指定时间执行。
- 常见场景：
  - 搜索输入联想
  - 输入框校验
  - window resize
  - 按钮重复点击控制
- 基本实现思路：
  - 使用定时器
  - 每次触发先清除上一次定时器
  - 到达等待时间后再执行目标函数
- 进阶点：
  - 保留 `this`
  - 透传参数
  - 支持立即执行（leading）
  - 支持取消（cancel）
- 节流含义：
  - 高频触发时，按照固定时间间隔执行
- 区别：
  - 防抖更关注“只执行最后一次”
  - 节流更关注“固定频率执行”

# 追问
- 如何实现“立即执行一次，之后防抖”的版本？
- `debounce` 在 React 组件里使用时要注意什么？
- 为什么有些场景更适合节流而不是防抖？
- 如果要给 `debounce` 增加 `cancel` 和 `flush` 方法，你会怎么设计？

# 参考答案
防抖的核心思想是：如果一个事件被频繁触发，那么只在“停止触发一段时间后”再执行一次函数。也就是说，前面的多次触发都会被取消，最终只保留最后一次。

一个基础版 `debounce` 可以这样写：

```js
function debounce(fn, delay) {
  let timer = null;

  return function (...args) {
    const context = this;
    clearTimeout(timer);
    timer = setTimeout(() => {
      fn.apply(context, args);
    }, delay);
  };
}
```

这个实现里，每次触发返回函数时，都会先清掉旧定时器，再开启一个新定时器。只有在 `delay` 时间内没有再次触发，`fn` 才会真正执行。

它常用于搜索框输入联想、表单校验、窗口 resize 等场景，因为这些操作如果每次触发都立刻执行，可能会带来性能浪费。

防抖和节流的区别是面试高频点。防抖是“只关心最后一次”，适合用户输入结束后再处理；节流是“按固定频率执行”，适合滚动、拖拽这类持续触发但又希望定期响应的场景。

如果要再进阶一点，面试里可以补充：一个完整的 `debounce` 通常还会支持立即执行、取消定时器以及手动立刻执行等能力。但基础版只要能正确处理定时器、`this` 和参数传递，通常就已经及格了。

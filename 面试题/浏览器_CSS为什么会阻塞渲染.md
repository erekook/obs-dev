# 题目
为什么 CSS 会阻塞渲染？

# 你的回答
因为要先拿到 css 的位置和大小信息，否则会导致页面元素闪烁。

# 面试点评
方向对，但表达不够标准。CSS 阻塞渲染的根因不是单纯“拿位置和大小”，而是浏览器必须先拿到最终样式信息，才能生成 Render Tree，进而继续布局和绘制。页面闪烁是结果，不是根因。

# 回答要点
- 浏览器渲染需要 DOM 和 CSSOM 一起生成 Render Tree
- 如果 CSS 未加载和解析完成，就无法确定元素最终样式
- 无法正确进行布局和绘制
- 如果不等待 CSS 就渲染，页面可能出现样式跳变或闪烁

# 参考答案
CSS 会阻塞渲染，是因为浏览器在构建 Render Tree 时需要同时拿到 DOM 和 CSSOM。如果 CSS 还没下载和解析完成，浏览器就无法确定元素的最终样式，也就没法继续进行后续的布局和绘制。
如果不等 CSS 就直接渲染，页面可能会先以错误样式显示，之后再重新计算和重绘，用户看到的就是闪烁或者样式跳变。
所以可以简单理解为，CSS 会影响 Render Tree，而 Render Tree 又直接影响布局和绘制，因此 CSS 会阻塞渲染。

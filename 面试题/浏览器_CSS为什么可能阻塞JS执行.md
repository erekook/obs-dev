# 题目
为什么 CSS 可能会阻塞 JS 执行？

# 你的回答
因为浏览器在执行某些 JS 之前，通常需要确保相关 CSS 已经加载完成，因为 JS 会读取元素样式。

# 面试点评
这次回答很准确，已经达到可直接用于面试的水平。继续保持这种“原因 + 影响”的表达方式会更稳。

# 回答要点
- JS 可能读取元素的计算样式
- 比如 getComputedStyle 等操作依赖 CSSOM 完整性
- 如果 CSS 未完成，样式结果不准确
- 为保证 JS 执行结果正确，浏览器通常会等相关 CSS 加载完成

# 参考答案
因为 JavaScript 可能会读取元素的计算样式，比如 `getComputedStyle`，也可能根据样式结果去操作 DOM。
如果这时 CSS 还没有加载和解析完成，浏览器就无法拿到准确的样式信息。
为了保证 JavaScript 执行结果正确，浏览器通常会先等相关 CSS 加载完成，再执行这段 JS。

# 追问
- 这是不是说明 CSS 一定会阻塞所有 JS？

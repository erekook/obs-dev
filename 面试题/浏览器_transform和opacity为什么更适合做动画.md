# 题目
为什么 transform 和 opacity 更适合做动画？

# 你的回答
这题当时不会。

# 面试点评
这是性能优化里的高频题，核心答案不是单纯“GPU 加速”，而是它们通常不会触发回流，更多只影响合成阶段。

# 回答要点
- transform 和 opacity 通常不会触发回流
- 不会影响文档流中的布局
- 很多情况下只需要走合成阶段
- 性能开销更小，动画更流畅
- GPU 合成是加分点，但不是唯一核心答案

# 参考答案
`transform` 和 `opacity` 更适合做动画，因为它们通常不会触发回流。`transform` 改变的是元素的变换效果，比如位移、缩放、旋转，但不会影响文档流中的布局；`opacity` 改变的是透明度，也不会影响元素尺寸和位置。
所以这类动画通常只需要走合成阶段，不需要重新布局和大面积重绘，性能开销更小，更容易保持流畅。在很多浏览器里，这类属性也更容易交给 GPU 合成层处理，因此动画体验通常更好。

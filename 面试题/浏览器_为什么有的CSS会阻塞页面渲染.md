# 题目
为什么有些 CSS 会阻塞页面渲染？浏览器为什么不先把内容显示出来，再慢慢应用样式？

# 回答要点
- 浏览器通常会把 CSS 当作渲染关键资源。
- 原因：
  - 页面渲染前需要知道元素最终样式
  - 否则先显示再改样式会导致闪烁和布局跳动
- CSS 对渲染的影响：
  - 构建 CSSOM
  - 与 DOM 结合生成渲染树
  - 决定布局和绘制结果
- 所以常说：
  - CSS 不一定阻塞 DOM 解析
  - 但会阻塞后续渲染
- 面试里可补充：
  - 非关键 CSS 可延后加载
  - 关键 CSS 可内联以优化首屏

# 追问
- CSS 阻塞的是 DOM 解析还是页面渲染？
- 为什么“先裸奔再套样式”通常不是好体验？
- 什么是关键 CSS？
- 如何减少 CSS 对首屏渲染的阻塞影响？

# 参考答案
有些 CSS 会阻塞页面渲染，核心原因是：浏览器在真正把页面画出来之前，需要先知道元素最终应该长什么样。

页面渲染不是只靠 HTML 就能完成的。HTML 解析后得到 DOM，但浏览器还需要拿到 CSS 规则，构建 CSSOM，再把 DOM 和 CSSOM 结合起来生成渲染树，之后才能做布局和绘制。

如果浏览器在 CSS 还没准备好时就急着把页面显示出来，用户可能会先看到一版“没样式或样式不完整”的页面，等 CSS 下载解析完成后，页面又突然跳一下、变一下。这种现象通常会带来闪烁、错位和糟糕体验。

所以浏览器宁可在关键 CSS 没准备好时稍微等等，也不愿意让用户先看到一版错误或不稳定的界面。这就是为什么很多 CSS 被认为是“渲染阻塞资源”。

更准确地说，CSS 通常不一定阻塞 HTML 的继续解析，但它会阻塞页面最终的可视化渲染流程。因为没有完整样式信息，浏览器很难安全地完成布局和绘制。

这也是前端性能优化里经常提到“关键 CSS”的原因。首屏真正需要的样式可以尽量优先提供，甚至内联；而首屏之外的非关键 CSS，则可以延后加载，减少首屏渲染等待。

面试里如果总结，可以说：CSS 会阻塞渲染，是因为浏览器在绘制页面前需要先确定样式、布局和渲染树；比起让用户先看到一版错误样式的页面，浏览器更倾向于等关键 CSS 准备好再稳定地渲染出来。

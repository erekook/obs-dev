# 题目
从浏览器地址栏输入一个 URL 到页面最终渲染出来，中间大致会经历哪些过程？请按顺序尽量完整地说一下。

# 回答要点
- 用户输入 URL，浏览器先进行合法性判断、补全协议、检查缓存或历史记录。
- 解析 URL，确定协议、域名、端口、路径、查询参数等。
- DNS 解析，把域名转换成 IP 地址。
- 建立连接：
  - TCP 三次握手
  - HTTPS 还需要 TLS 握手、证书校验、密钥协商
- 浏览器发送 HTTP 请求，请求 HTML 文档。
- 服务端处理请求并返回响应报文。
- 浏览器接收响应后，根据状态码、响应头决定如何处理，比如缓存、重定向、内容类型识别。
- 开始解析 HTML，构建 DOM 树。
- 解析 CSS，构建 CSSOM。
- DOM + CSSOM 合成 Render Tree。
- 布局（Layout / Reflow）：计算元素几何信息。
- 绘制（Paint）：把各个节点绘制成像素。
- 合成（Composite）：图层合成并显示到屏幕。
- 解析过程中如果遇到 JS：
  - 可能阻塞 HTML 解析
  - JS 可操作 DOM、CSSOM，影响后续渲染
- 页面加载后还可能继续请求图片、字体、异步数据等资源。

# 追问
- 为什么 CSS 会阻塞渲染，而 JS 可能阻塞解析？
- `defer` 和 `async` 的区别是什么？
- 回流和重绘分别是什么，哪些操作更贵？
- 浏览器缓存会在哪些阶段生效？

# 参考答案
当用户在浏览器地址栏输入一个 URL 并回车后，浏览器首先会对输入内容做解析，比如补全协议、判断是搜索词还是合法网址，然后拆分出协议、域名、端口、路径等信息。

接着浏览器会尝试获取目标服务器地址，这一步通常需要 DNS 解析，把域名转换成 IP。拿到 IP 后，浏览器与服务器建立连接。如果是 HTTP，一般是 TCP 三次握手；如果是 HTTPS，还会额外进行 TLS 握手、证书校验和密钥协商。

连接建立后，浏览器发起 HTTP 请求，服务器返回响应内容。浏览器收到响应后，会先根据状态码和响应头做一些处理，比如是否重定向、是否命中缓存、内容类型是什么。

如果返回的是 HTML，浏览器就开始解析 HTML，逐步构建 DOM 树。同时遇到 CSS 会下载并解析，形成 CSSOM。DOM 和 CSSOM 结合后生成 Render Tree，再进入布局阶段，计算每个元素的位置和尺寸，最后进入绘制和图层合成，把页面显示出来。

在这个过程中，JavaScript 很关键。因为 JS 可能修改 DOM 和样式，所以默认情况下普通 `script` 会阻塞 HTML 解析，浏览器通常要等它执行完再继续往下解析。CSS 也会影响最终样式计算，所以在某些场景下会影响渲染时机。

另外，页面首次渲染出来并不代表流程完全结束。图片、字体、懒加载模块、接口数据等资源可能还在继续加载，用户看到的页面也可能随着异步数据返回而更新。

如果要一句话总结，这个过程可以概括为：输入 URL → 解析域名并建立连接 → 发起请求并接收响应 → 解析 HTML/CSS/JS → 构建渲染树 → 布局、绘制、合成 → 页面显示。

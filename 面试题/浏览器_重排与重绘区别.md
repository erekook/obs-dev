# 题目
请你讲一下浏览器中的重排（Reflow / Layout）和重绘（Repaint）。它们分别在什么情况下发生？为什么通常说重排的成本更高？

# 回答要点
- 重排是重新计算元素的几何信息和布局位置。
- 重绘是不改变布局前提下，重新绘制元素外观。
- 常见触发重排的情况：
  - 元素尺寸变化
  - 位置变化
  - DOM 增删
  - 字体大小变化
  - 浏览器窗口尺寸变化
  - 读取某些会强制刷新布局的属性后再修改样式
- 常见触发重绘的情况：
  - 颜色变化
  - 背景变化
  - 可见性变化（不影响布局时）
- 为什么重排更贵：
  - 会影响布局树
  - 可能引起子元素、父元素甚至相邻元素重新计算
  - 重排之后通常还会伴随重绘
- 优化思路：
  - 批量修改样式
  - 避免频繁读写布局属性交替执行
  - 使用 `transform`、`opacity` 做动画
  - 脱离文档流后再集中操作
  - 减少复杂 DOM 结构上的高频改动

# 追问
- 哪些属性读取会导致强制同步布局？
- 为什么 `transform` 动画通常比 `top/left` 更高效？
- `display: none` 和 `visibility: hidden` 对布局和绘制有什么影响？
- 浏览器的合成层和性能优化之间有什么关系？

# 参考答案
浏览器渲染页面时，大致要经历样式计算、布局、绘制和合成等阶段。重排和重绘就是其中两个常见的性能概念。

重排，也叫 Layout 或 Reflow，指的是浏览器需要重新计算元素的几何信息，比如位置、宽高、盒模型尺寸，以及元素和周围节点之间的布局关系。只要页面结构或尺寸相关信息发生变化，就可能触发重排。

比如修改元素宽高、修改字体大小、增删 DOM 节点、改变窗口大小，或者让元素从 `display: none` 变成可见，通常都会引发重排。

重绘指的是元素的几何位置没有变化，但外观需要重新绘制，比如颜色、背景色、阴影等样式变化。这类操作不需要重新计算布局，只需要重新把元素画出来。

通常说重排比重绘更贵，是因为重排影响范围更大。浏览器不仅要重新计算当前元素，还可能影响它的子元素、父元素，甚至相邻元素的布局关系。而且一旦发生重排，后面通常还要继续执行重绘，所以它的成本更高。

前端优化里，一个常见原则是避免频繁触发布局计算。比如不要在循环中交替读取 `offsetWidth` 这类布局属性再修改样式，因为这容易造成强制同步布局。动画场景下，更推荐使用 `transform` 和 `opacity`，因为它们通常不会触发布局重算，性能更稳定。

总结就是：重排是“重新算位置和大小”，重绘是“重新画外观”。重排通常比重绘代价更高，而且很多重排最终还会连带触发重绘，所以性能优化时要优先减少重排。
